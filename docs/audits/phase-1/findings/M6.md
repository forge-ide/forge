## Scope

Area: `crates/forge-session`. Threat class: **T6 — UDS / local symlink race**. A classic `/tmp` race that is non-exploitable when the parent directory is 0700, but becomes exploitable under the H8 fallback to a shared `/tmp/forge-0/` directory.

## Finding

- **Location:** `crates/forge-session/src/server.rs:68-71`
- **Severity:** medium
- **Threat class:** T6

`serve_with_session` does:

```
if path.exists() { remove_file(path).await?; }
listener.bind(path)
```

Between `exists()`, `remove_file`, and `bind`, an attacker with write access to the parent directory can race:
1. Pre-create the socket path as a regular file or symlink.
2. Wait for the daemon's `remove_file` to delete it.
3. Re-create as a symlink to `/tmp/attacker-sock`; the subsequent `bind` resolves the symlink.

Even the benign case leaks the pattern of when a daemon is starting. Under H8's `/tmp/forge-0/` fallback, this is exploitable; under `XDG_RUNTIME_DIR` (0700), non-exploitable.

## Reproduction

Requires a shared `/tmp/forge-0/` (see H8).

1. Attacker pre-creates `/tmp/forge-0/forge/sessions/<predicted-id>.sock` as a symlink.
2. Victim invokes `forge-cli` which spawns `forged` with a matching `FORGE_SESSION_ID`.
3. `remove_file` unlinks the symlink; `bind` succeeds on the attacker's target.

## Remediation

Try `bind` first without `remove_file`. On `EADDRINUSE`, do a short `UnixStream::connect` with a timeout to check whether the socket is live. If unreachable (orphan), only then `unlink` and retry. Never blindly `remove_file` in a world-writable parent.

Preferred: require `XDG_RUNTIME_DIR` (0700) as H8's remediation — removes the exploit window entirely.
