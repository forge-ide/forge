## Scope

Area: `crates/forge-session`. Threat class: **T6 — UDS socket mode**. Phase 1's IPC trust boundary is the Unix domain socket: anyone who can connect to it can drive the session, approve tool calls, and read all events. Socket permissions are the *only* thing keeping co-located local processes out.

## Finding

- **Location:** `crates/forge-session/src/main.rs:117-126` (`resolve_socket_path`), `crates/forge-session/src/server.rs:65-71` (listener bind)
- **Severity:** high
- **Threat class:** T6

No code sets socket or parent-directory permissions — grep for `set_permissions`, `PermissionsExt`, `mode(0o`, or `umask` across the crate yields zero hits. When `XDG_RUNTIME_DIR` is unset, `resolve_socket_path` falls back to `/tmp/forge-{uid}` where **`uid` is read from `std::env::var("UID")`** (a shell variable, not generally exported into child processes) with fallback `"0"`. For most spawn paths `UID` is absent and the literal string `"0"` is used regardless of the daemon's real uid — so multiple local users share `/tmp/forge-0/forge/sessions/`.

`tokio::fs::create_dir_all` creates with default permissions (`0777 & ~umask`, typically `0755`). `UnixListener::bind` creates the socket with `0777 & ~umask`, typically `0755` — world-connectable.

Any local user can connect, complete the Hello/Subscribe handshake, send `SendUserMessage` to drive the victim's model, approve their own tool calls, and exfiltrate events.

## Reproduction

1. Start `forged` as user alice with `XDG_RUNTIME_DIR` unset and `UID` unset in environment.
2. As user bob (same machine), observe `/tmp/forge-0/forge/sessions/` is readable+writable; the `.sock` has mode `0755`.
3. As bob: `socat - UNIX-CONNECT:/tmp/forge-0/forge/sessions/<id>.sock` then speak the frame protocol — send `Hello` + `Subscribe` + `SendUserMessage`; observe alice's session events on the read side.

## Remediation

1. In `resolve_socket_path`, use `nix::unistd::getuid()` (or `libc::getuid()`) instead of `std::env::var("UID")`.
2. Create the parent directory explicitly with mode `0700` if it does not already exist.
3. After `bind`, chmod the socket to `0600` via `tokio::fs::set_permissions` + `PermissionsExt::from_mode(0o600)`.
4. **Preferred:** reject the fallback entirely — require `XDG_RUNTIME_DIR` and refuse to start otherwise, with a clear error surfacing the missing prerequisite.
