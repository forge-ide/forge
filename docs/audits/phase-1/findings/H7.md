## Scope

Area: `crates/forge-session`. Threat class: **T1 (approval-gate fidelity) / T2 (allowed_paths bypass)**. Phase 1 introduces workspace-scoped filesystem tools; the whole point of the `allowed_paths` boundary is to stop an LLM from reaching outside the workspace after one approval. If the boundary is `**`, the approval UI is the only line of defense — one click-through to `fs.read /home/alice/notes.md` followed by a model that next emits `fs.read ~/.ssh/id_rsa` has only a second approval prompt to stop it.

## Finding

- **Location:** `crates/forge-session/src/server.rs:313`
- **Severity:** high
- **Threat class:** T1 / T2

`run_turn` is invoked with `allowed_paths = vec!["**".to_string()]`. The `glob::Pattern::new("**").matches_path()` method returns `true` for **every absolute path** (verified against the `glob` crate directly): `/etc/passwd`, `/root/.ssh/id_rsa`, `/home/$USER/.ssh/id_rsa`, `/var/log/**`, `~/.aws/credentials`, etc. `fs.read`, `fs.write`, `fs.edit` all validate against this allow-list — which validates nothing.

## Reproduction

1. Start `forged` (or a test harness) with any workspace.
2. From a connected UI, emit a tool call `fs.read {"path": "/etc/passwd"}`.
3. Approval gate fires with preview `Read file /etc/passwd`.
4. Click approve → `ToolCallCompleted` carries `/etc/passwd` contents in its result.

Or: approve one legitimate-looking `fs.read` → model follows with `fs.read ~/.ssh/id_rsa` → second approval is the only remaining defense.

## Remediation

Compute `allowed_paths` from `workspace_path` at session setup: if `Some(ws)`, pass `[format!("{}/**", ws.display())]` (after canonicalization). If no workspace, reject `fs.*` tool calls entirely.

Optional hardening: add a permanent deny-list for `~/.ssh/**`, `~/.aws/**`, `~/.config/**` containing credentials, `/etc/shadow`, etc.

Longer-term: pass `workspace_root` into `ToolCtx` and have `forge-fs` enforce a root directly, rather than a glob pattern that happens to match nothing.
