## Scope

Area: `web/packages/app`. Threat class: **T7 — deserialization of Tauri event payloads**. The frontend types `SessionEventPayload.event` as `unknown` (good) but then `as string`-casts individual fields without narrowing (bad). Breaks silently on version skew; compounds any future defect in the bridge writer.

## Finding

- **Location:** `web/packages/app/src/ipc/events.ts:17-85`, `web/packages/app/src/ipc/session.ts:17-21`
- **Severity:** medium
- **Threat class:** T7

`fromRustEvent` indexes arbitrary fields and type-asserts them:
- `message_id: ev['id'] as string`
- `text: ev['text'] as string`
- `delta: ev['delta'] as string`
- `tool_name: ev['tool'] as string`
- `args_json: JSON.stringify(ev['args'] ?? null)`
- `preview: ev['preview'] as { description: string }`

No `typeof` check. If a malformed daemon payload arrives (bug, version skew, compromised bridge writer), a field that should be a string can be `undefined`, a number, an object, or an array, and flows into the store typed as `string`.

Practical risk today is UI breakage, not compromise — ChatPane's text interpolation is Solid-escaped. But the threat model explicitly flags this entry point.

## Reproduction

Emit `{"type":"assistant_delta","id":"m1"}` (omit `delta`) — the adapter returns `{ kind:'AssistantDelta', message_id:'m1', delta: undefined as any }`; the messages store concatenates `undefined` via `turn.text += event.delta`, rendering the literal string `"undefined"`.

Emit `{"type":"tool_call_approval_requested","id":"t1"}` without `preview` — `ApprovalPrompt` reads `props.preview.description` on an undefined object and throws at render time.

## Remediation

Add narrowing helpers (`isString`, `isObjectWith(shape)`) in `ipc/events.ts` and return `null` from `fromRustEvent` for any event where a required field is missing / wrong-typed. Log once at `warn` so drift surfaces.

Tighten the type of `SessionEventPayload.event` from `unknown` to a discriminated union once `forge_core::Event` gets a TS codegen pass (the generated types in `web/packages/ipc/src/generated/` already show the pattern).
