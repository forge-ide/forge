## Scope

Area: `crates/forge-shell`. Threat class: **T7 — Tauri command deserialization / SSRF-equivalent against UDS**. A webview with code-execution (the CSP-null threat model) can redirect the host process to connect to any Unix domain socket the shell's uid can reach.

## Finding

- **Location:** `crates/forge-shell/src/ipc.rs:44-55`, `crates/forge-shell/src/bridge.rs:107-154`
- **Severity:** high
- **Threat class:** T7

`session_hello(session_id, socket_path: Option<String>, ...)` converts `socket_path` into a `PathBuf` and passes it straight to `UnixStream::connect(&path)`. No allowlist check, no canonicalization, no symlink rejection.

A webview caller (any gadget given CSP null) can direct the shell to connect to any UDS on the host the shell's uid can reach:
- `$SSH_AUTH_SOCK` (ssh-agent)
- Docker control socket if the user has Docker permissions
- A `forged` socket for a different workspace
- A trap socket placed by a co-located malicious process that speaks a fake `HelloAck`

The shell immediately writes a `Hello` frame (leaking PID and `$USER`) and reads a frame back. Whatever bytes the peer returns must parse as `HelloAck`, but the peer controls the payload — `session_id` string, `workspace` string — which then flow into the webview event stream.

## Reproduction

1. From a webview (requires CSP-null + no per-window authz): `await window.__TAURI_INTERNALS__.invoke('session_hello', { sessionId: 'x', socketPath: '/run/user/1000/ssh-agent.socket' })`.
2. Shell writes the `Hello` JSON frame to ssh-agent (which rejects it, but the probe succeeded).

Directed attack: start a rogue listener at `/tmp/rogue.sock` that returns a crafted `HelloAck`; invoke `session_hello` with that path; crafted `session_id`/`workspace` reach the webview as the ack response.

## Remediation

**Preferred:** drop the `socket_path` parameter entirely from the public command. Always use `default_socket_path(session_id)`. The test path (`tests/ipc_commands.rs:77`) is the only current consumer of the override and can use a test-only constructor (e.g. a `#[cfg(test)]` field on `BridgeState`).

**Alternative:** validate that the supplied path canonicalizes under the expected `<base>/forge/sessions/` directory and has no `..` components.

Combine with the per-window authz fix (H10) so only the matching Session window can even call `session_hello` for its own id.
