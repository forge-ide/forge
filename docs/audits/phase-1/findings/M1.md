## Scope

Area: `crates/forge-cli`. Threat class: **T12 — raw-PID SIGTERM (path traversal vector)**. Compounds H1/H2 by providing an attacker-controlled channel to select *which file* `session_kill` reads the pid from.

## Finding

- **Location:** `crates/forge-cli/src/main.rs:184`, `crates/forge-cli/src/socket.rs:4-13, 27-29`
- **Severity:** medium
- **Threat class:** T12a (raw-PID signal input validation)

`session_kill(id)` and `session_tail(id)` take the session id string straight from the CLI and pass it to `forge_cli::socket::pid_path` / `socket_path`, which does `base.join(format!("{session_id}.sock"))` then `.with_extension("pid")`. No validation that the id is hex / matches `SessionId` format (`[0-9a-f]{16}`).

A session id like `../../../home/victim/some.pid` causes `pid_path` to resolve outside the sessions dir. `with_extension("pid")` only constrains the final extension, so an attacker-controlled or typo'd id can make the CLI read an arbitrary `.pid`-extensioned file the user has access to, parse its first line as a pid, and pass it to `libc::kill`.

Combined with H1 (no pid<=0 check) and H2 (no ownership check): a crafted id pointing to any file whose first line parses as a pid is a kill primitive on an attacker-chosen process.

## Reproduction

```bash
echo -n "$$" > /tmp/evil.pid
forge session kill '../../tmp/evil'
# SIGTERM sent to current shell
```

Or with id `../../tmp/evil` where `/tmp/evil.pid` contains `0` or `-1`, achieve the H1 effect through path traversal rather than a planted `.pid` file.

## Remediation

Validate the session id in CLI parsing — clap `value_parser` that checks `^[0-9a-f]{16}$` (or whatever `SessionId`'s canonical format is). Reject `/`, `..`, or `.` in the id.

Optionally expose `forge_cli::socket::session_id_is_valid(&str) -> bool` and gate every path-building function on it.
