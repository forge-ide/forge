## Scope

Area: `crates/forge-session`. Threat class: **T3 / T4 — resource**. Combined with H6 (orphan on timeout), an infinite timeout extends the window during which an orphaned grandchild is still registered as the current tool call.

## Finding

- **Location:** `crates/forge-session/src/tools/shell_exec.rs:97-100`
- **Severity:** low
- **Threat class:** T3 / T4

`timeout_ms` is taken verbatim from args with default `30000`. A provider can pass `u64::MAX`. CPU rlimit still caps compute at 30s per process, but a shell backgrounding sleeps can hold the tool-call future open indefinitely.

## Reproduction

```json
{"timeout_ms": 18446744073709551615, "command": "/bin/sh", "args": ["-c", "sleep 9999"]}
```

## Remediation

Clamp `timeout_ms` to a configurable ceiling (e.g. `10 * 60 * 1000` ms = 10 minutes).
