## Scope

Area: `crates/forge-session`. Threat class: **T4 — sandbox escape**. The Level 1 sandbox uses `setsid`, `setpgid`, and a `ChildRegistry` that tracks process groups so `forged` can `killpg` every subprocess at shutdown. If a process leaks out of both the registry and the `Drop` path, the sandbox no longer contains it — the whole point of L1 isolation defeated by an orphan.

## Finding

- **Location:** `crates/forge-session/src/tools/shell_exec.rs:124-174` combined with `crates/forge-session/src/sandbox.rs:264-270`
- **Severity:** high
- **Threat class:** T4

`run_linux` calls `sb.spawn()` to get a `SandboxedChild`, then immediately `sandboxed.into_child()`. `into_child()` does two things: (1) sets `released = true` so `SandboxedChild::drop` is a no-op; (2) calls `registry.remove(pgid)` removing the pgid from the session-scoped `ChildRegistry`. The returned `tokio::process::Child` is then moved into a future passed to `tokio::time::timeout`. `kill_on_drop` is never set (verified `grep -r kill_on_drop` = zero matches).

When the timeout fires, the `Child` is dropped without sending any signal — tokio's default is to orphan the process. Because the pgid has already been removed from `ChildRegistry`, `child_registry.kill_all()` on daemon shutdown cannot clean it up either. The sandbox process group outlives the daemon.

## Reproduction

1. Connect to a running `forged`.
2. Trigger a tool call via the mock provider: `shell.exec {"command": "/bin/sh", "args": ["-c", "(sleep 3600 </dev/null >/dev/null 2>&1 &); echo fast"], "timeout_ms": 100}`.
3. Approve.
4. `shell.exec` returns `{"error": "timeout after 100ms"}`.
5. `ps -o pid,pgid,cmd | grep sleep` shows the `sleep` still running.
6. `SIGTERM forged`; the `sleep` keeps running — it was not in the registry at shutdown.

## Remediation

Preferred: keep the `SandboxedChild` wrapper live through the wait — use `.as_child_mut()` to read stdout/stderr/wait. Never call `.into_child()` in the normal flow. `Drop` then still `killpg`s and the registry entry persists.

Minimal: in the timeout-error branch of `shell_exec.rs`, explicitly call `killpg(pgid, SIGKILL)` before returning, using the pgid captured *before* `into_child()`.

Preferred is strictly better because it covers panics and task cancellations, not just the timeout path.
