# Generation Pipelines

> Two CI-enforced pipelines that keep generated artifacts in sync with their
> sources of truth: design tokens (CSS) and IPC types (TypeScript bindings
> from Rust).

The two pipelines are independent but share a pattern: a Rust- or
docs-side source of truth, a script that emits the generated form, and a
CI gate that fails the build on drift. If you touched a Rust IPC type or a
design token, you almost certainly need to regenerate.

---

## 1. Design tokens (CSS)

> Extracted from IMPLEMENTATION.md §9.4 — design token drift check and CI enforcement

- `web/packages/design/src/tokens.css` is the single source of CSS vars
- Checked against `docs/design/token-reference.md` by a CI script
  (`scripts/check-tokens.mjs`) that parses both and fails on drift
- Component files never use raw hex — lint rule enforces `var(--color-*)`

### Usage in component code

```css
/* Correct — use tokens */
color: var(--color-text-primary);
background: var(--color-surface-2);
border-color: var(--color-border-brand);

/* Wrong — never raw hex in component code */
color: #eae6de;
background: #13161d;
```

### CI check

`scripts/check-tokens.mjs` runs as a CI step. It:
1. Parses token definitions from `web/packages/design/src/tokens.css`
2. Parses the token reference table in `docs/design/token-reference.md`
3. Fails the build if any token drifts between the two sources

Run it locally with `pnpm --filter forge-web run check-tokens` (or
`node scripts/check-tokens.mjs`). Fix drift by updating `tokens.css` to
match the reference doc (the doc is authoritative) or, if intentional,
updating both simultaneously.

## 2. ts-rs Rust to TypeScript IPC types

Rust IPC types in `crates/forge-core` are the source of truth for the
TypeScript bindings used by the web frontend. The
[`ts-rs`](https://github.com/Aleph-Alpha/ts-rs) crate emits one `.ts`
file per annotated type into `web/packages/ipc/src/generated/`. CI fails
the build if the committed bindings drift from the Rust definitions.

### What's wired up

- **Crate dependency:** `ts-rs = "12"` in `crates/forge-core/Cargo.toml`
- **Output directory:** `web/packages/ipc/src/generated/` (currently 12
  files: `AgentId`, `AgentInstanceId`, `ApprovalScope`, `CompactTrigger`,
  `MessageId`, `ProviderId`, `RosterScope`, `SessionId`,
  `SessionPersistence`, `SessionState`, `ToolCallId`, `WorkspaceId`)
- **Annotation pattern** (in `crates/forge-core/src/types.rs` and
  `src/ids.rs`):

  ```rust
  use serde::{Deserialize, Serialize};
  use ts_rs::TS;

  #[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, TS)]
  #[ts(export, export_to = "../../../web/packages/ipc/src/generated/")]
  pub enum SessionState {
      Active,
      Archived,
      Ended,
  }
  ```

  Each generated file carries the header
  `// This file was generated by ts-rs. Do not edit this file manually.`

### How regeneration is triggered

The `#[ts(export)]` attribute makes ts-rs synthesize a `#[cfg(test)]`
test (e.g. `export_bindings_session_state`) that writes the `.ts` file
to disk. Running `cargo test --workspace` therefore regenerates every
binding — there is no separate `xtask` or build script.

```bash
cargo test --workspace
git status web/packages/ipc/src/generated/   # any diff = your change
git add web/packages/ipc/src/generated/
```

If `cargo test` does not update a file you expect to change, the type
either lacks `#[derive(TS)]` and `#[ts(export, export_to = ...)]`, or
the export path on the derive is wrong.

### CI gate

`.github/workflows/ci.yml` runs `cargo test --all` to populate the
generated directory, then enforces drift with:

```yaml
- name: Check TS bindings are up to date
  run: git diff --exit-code web/packages/ipc/src/generated/
```

A non-empty diff fails the build. The fix is always: run
`cargo test --workspace` locally, commit the regenerated `.ts` files,
and push.

### Workflow when changing a Rust IPC type

1. Edit the Rust type (or add a new one with `#[derive(..., TS)]` and
   `#[ts(export, export_to = "../../../web/packages/ipc/src/generated/")]`).
2. Run `cargo test --workspace`.
3. `git diff web/packages/ipc/src/generated/` — confirm the change
   looks right.
4. Commit the regenerated `.ts` files alongside the Rust change.

### Troubleshooting

- **Generated file is stale or missing a variant** — confirm
  `#[derive(TS)]` is on the type and `#[ts(export, export_to = ...)]`
  is present. Then `cargo test --workspace`. If it still does not
  update, `cargo clean -p forge-core && cargo test --workspace`.
- **CI fails on the drift check but local diff is empty** — you
  probably forgot to `git add` the regenerated files. Generated files
  are committed; they are not produced on the CI runner from scratch.
- **Newtype wrappers (e.g. `SessionId(String)`)** — declare them via
  the `id_newtype!` macro in `crates/forge-core/src/ids.rs`, which
  applies `#[ts(export, ..., type = "string")]` so the binding is
  emitted as `export type SessionId = string;` rather than an opaque
  wrapper.
